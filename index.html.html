AI Foundation Agents
1
Data Discovery & Ingestion
Agent scans source systems and ingests data inventory
▼
The foundation layer. Connects to all source systems via the platform's native connectors, continuously scans for new and modified data assets, builds and maintains a live data inventory, and passes all discovered assets to Layer 2 for classification. MCP connects the AI agent to the privacy platform — source system connectors remain the platform's responsibility.
Agent type
Discovery Agent
Scan mode
Continuous + triggered on change events
Connector note
Platform owns source connectors; MCP links AI to platform only
Outputs
Live data inventory → Layer 2 classification queue
2
Classification & Labelling
Agent identifies and labels personal and sensitive data
▼
Applies automated classification to all data discovered by Layer 1. Identifies personal information, sensitive categories (health, biometric, financial), and special categories under GDPR. Assigns regulatory jurisdiction tags, confidence scores, and sensitivity labels. All classification feeds directly into every operational layer above.
Agent type
Classification Agent
Categories
PI · Sensitive PI · Special category · De-identified
Jurisdictions
AU · UK/EU · US · JP labelling schemas
Outputs
Labelled data inventory, confidence scores, jurisdiction tags
│
AI Operational Specialist Agents
3
Consent Management
Agent captures, records, and enforces consent preferences
▼
Operates the consent platform: records individual consent and preference signals with timestamped evidence, propagates consent decisions downstream to processing systems, monitors for consent drift (expired, revoked, or conflicting), and ensures processing activities only proceed where valid consent or another lawful basis applies.
Agent type
Consent Orchestration Agent
Propagates to
Layers 4, 6, 7 — enforces downstream
Evidence standard
Timestamped, version-locked, tamper-evident
Outputs
Consent records, preference profiles, propagation logs
4
Cookie & Tracking Management
Agent scans, classifies, and enforces tracking compliance
▼
Continuously scans web properties to discover tracking technologies, classifies cookies and pixels by purpose and necessity, validates consent banner configurations against ePrivacy and GDPR requirements, blocks non-consented trackers, and reports compliance status to Layer 8.
Agent type
Tracking Compliance Agent
Scope
Web, app, pixel, tag manager
Frameworks
ePrivacy, GDPR, CCPA opt-out
Outputs
Cookie inventory, banner config, block decisions
5
Privacy Notice Management
Agent drafts, versions, and publishes compliant notices
▼
Monitors changes to data flows, processing purposes, and applicable regulations. When triggered, drafts updated privacy notice content, versions and date-stamps documents, coordinates publication across digital properties, and maintains a complete version history for regulatory evidence.
Agent type
Notice Management Agent
Triggers
Processing changes, regulatory updates, new data flows
Jurisdictions
AU · UK · EU · US (state) · JP
Outputs
Versioned notices, publication records, change log
6
DSR Management
Agent receives, routes, fulfils, and closes data subject requests
▼
Manages the full lifecycle of Data Subject Requests — access, erasure, rectification, portability, objection. Ingests requests from intake channels, verifies identity, queries source systems via platform connectors, assembles response packages, tracks statutory deadlines, and issues compliant responses without human intervention for standard requests.
Agent type
DSR Fulfilment Agent
Request types
Access, erasure, rectification, portability
Deadline tracking
AU 30-day · GDPR 30-day · extensions logged
Escalates when
Identity unclear, competing legal basis, complex erasure
7
Records Management & Retention
Agent enforces retention schedules and triggers disposal
▼
Maintains the Records of Processing Activities (RoPA) register, applies retention schedules mapped to data categories and jurisdictions, monitors data age, automatically flags records for review or disposal, and produces audit-ready evidence of compliant retention management.
Agent type
Records & Retention Agent
Inputs from
Classification (Layer 2), Consent (Layer 3)
Key artefact
RoPA — auto-maintained, audit-ready
Outputs
Retention triggers, disposal records, RoPA updates
8
Risk & Compliance Monitoring
Agent continuously scores risk and flags regulatory drift
▼
Continuously evaluates the privacy risk posture across all data flows, processing activities, and third-party relationships. Monitors for regulatory change, recalculates risk scores, and surfaces compliance gaps to Layer 12 for prioritisation and action.
Agent type
Risk Monitoring Agent
Runs
Continuously — event-driven and scheduled
Feeds into
Layers 9 (breach), 10 (assessments)
Outputs
Risk register, compliance gap report, drift alerts
9
Breach Notification & Response
Agent detects, triages, and notifies within obligation windows
▼
Monitors anomaly signals from the risk layer, detects potential data breaches, assesses notifiability against multi-jurisdiction thresholds (AU NDB scheme, GDPR 72hr), auto-drafts regulator and individual notifications, tracks obligation windows, and escalates critical incidents to human governance immediately.
Agent type
Incident Response Agent
Jurisdictions
AU (NDB), UK/EU (GDPR), US (state laws)
Triggers from
Anomaly signals (Layer 8)
Outputs
Incident record, notification drafts, countdown timers
10
Assessment Management
Agent drafts, tracks, and closes PIAs and DPIAs
▼
Monitors project and change registers, automatically triggers Privacy Impact Assessments and DPIAs based on risk criteria, drafts assessment content using classification data from lower layers, tracks completion deadlines, and flags overdue items to Layer 12.
Agent type
Assessment Agent
Frameworks
DPIA (GDPR), PIA (AU Privacy Act)
Triggers from
Risk scores (Layer 8), new data flows (Layer 1)
Outputs
Draft assessments, deadline alerts, completion status
│
AI Orchestration Layer
12
AI Agent Orchestration
Master orchestrator coordinates all specialist agents
▼
The orchestration brain of the platform. Receives high-level objectives from Layer 11 governance, decomposes them into tasks, assigns specialist agents across Layers 1–10, manages inter-layer data flow, and escalates decisions that exceed defined thresholds back to humans.
Who acts
Orchestrator AI Agent
Protocol
MCP — connects AI to privacy platform
Key functions
Task routing, agent coordination, threshold monitoring
Escalates when
Risk score, legal ambiguity, or confidence below threshold
↑ Escalates to Layer 11 ↓ Directs Layers 1–10
│
Human Governance Layer
11
Governance & Audit
Human approves, sets policy, reviews audit trail
▼
The only layer where humans are the primary actors. Privacy officers and legal counsel set policy rules, approve escalated agent decisions, review immutable audit logs, and maintain regulatory accountability that cannot be delegated to AI.
Who acts
Privacy Officer, Legal, DPO
Role type
Governance — not operational
Key functions
Policy setting, escalation approval, audit review
Outputs
Approved policies, decisions, signed-off audit records
↓ Escalations from Layer 12 ↑ Policy rules to Layer 12
Layer key
Layers 1–2 — AI foundation agents (discovery, classification)
Layers 3–10 — AI operational specialist agents
Layer 12 — AI orchestrator (coordinates all agents)
Layer 11 — Human governance (approves, sets policy)
